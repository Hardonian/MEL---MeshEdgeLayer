# Configuration truth guide

This guide documents only config keys that the current codebase reads or uses for operator-visible behavior.

## Top-level sections

### `bind`

- `api`: required listener for UI and JSON API.
- `metrics`: reserved field only; RC1 does not start a metrics listener.
- `allow_remote`: if `true`, validation requires auth unless `auth.allow_insecure_remote=true` is deliberately set.

### `auth`

- `enabled`: enables HTTP basic auth for UI and API.
- `session_secret`: validated and generated by `mel init`, but RC1 auth is basic-auth based rather than cookie-session heavy.
- `ui_user`, `ui_password`: basic auth credentials.
- `allow_insecure_remote`: unsafe override for deliberate development use.

### `storage`

- `data_dir`: required writable directory.
- `database_path`: SQLite file path.
- `encryption_key_env`: env var name checked when `encryption_required=true`.
- `encryption_required`: validation guard only in RC1; MEL does not encrypt SQLite at rest.

### `logging`

- `level`, `format`: accepted and stored in config, with JSON logging used by the daemon logger.

### `retention`

- `messages_days`, `telemetry_days`, `audit_days`, `precise_position_days`: consumed by validation, privacy/policy logic, and retention cleanup.

### `privacy`

- `store_precise_positions`
- `mqtt_encryption_required`
- `map_reporting_allowed`
- `redact_exports`
- `trust_list`

These affect linting, doctor summaries, privacy audit output, policy recommendations, and export behavior.

### `transports`

Supported live types:

- `serial`
- `tcp`
- `mqtt`

Recognized but not operator-supported in RC1:

- `serialtcp`
- `ble`
- `http`

### `features`

- `web_ui`: if false, MEL still serves JSON endpoints but does not register the HTML UI route.
- `metrics`: reserved only; no metrics server exists.
- `ble_experimental`: does not enable a working BLE transport.

### `rate_limits`

- `http_rps`: stored in config but not yet enforced by the web layer.
- `transport_reconnect_seconds`: used as default reconnect backoff for transports.

## Environment variables actually read

- `MEL_BIND_API`
- `MEL_BIND_METRICS`
- `MEL_BIND_ALLOW_REMOTE`
- `MEL_DB_PATH`
- `MEL_DATA_DIR`
- `MEL_AUTH_ENABLED`
- `MEL_SESSION_SECRET`
- `MEL_UI_USER`
- `MEL_UI_PASSWORD`
- `MEL_AUTH_ALLOW_INSECURE_REMOTE`
- `MEL_PRIVACY_STORE_PRECISE_POSITIONS`
- `MEL_PRIVACY_MAP_REPORTING_ALLOWED`
- `MEL_PRIVACY_MQTT_ENCRYPTION_REQUIRED`
- `MEL_RETENTION_MESSAGES_DAYS`

No other `MEL_*` environment variables should be treated as supported unless they are added to `internal/config/config.go`.

## Operator guidance

- Start with one transport.
- Use stable serial device paths on Linux/Pi.
- Keep `bind.allow_remote=false` unless you have a deliberate access design.
- Keep `privacy.redact_exports=true` by default.
- Treat metrics and BLE config knobs as placeholders, not proof of implementation.
