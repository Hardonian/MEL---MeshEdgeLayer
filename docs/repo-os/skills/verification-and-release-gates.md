# Skill: Verification and Release Gates

Use for any change that can alter operator truth, control behavior, privacy, or capability claims.

## Verification mapping
- [ ] Work is mapped to `docs/repo-os/verification-matrix.md` sections.
- [ ] Required tests/checks are run or explicitly waived with rationale.
- [ ] Claim-affecting docs are cross-checked against implementation.

## Release readiness checks
- [ ] `docs/repo-os/release-readiness.md` checklist reviewed.
- [ ] Capability/support claims align with tested behavior.
- [ ] Degraded/unknown semantics are explicit in operator surfaces.
- [ ] Canonical truth is not coupled to local inference runtime success.
- [ ] Residual risks and caveats are documented concretely.
