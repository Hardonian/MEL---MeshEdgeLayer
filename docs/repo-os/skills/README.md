# MEL Repo-Local Skills (Reusable Execution Modules)

These skills are repo-local execution modules for future agents/contributors. Use them to enforce MEL doctrine without repeating long prompts.

## Skill modules
1. `operator-truth-and-transport.md`
   - No-theatre truth checks, support-matrix boundaries, degraded-state honesty.
2. `precision-layering-and-scientific-observation.md`
   - Layered reasoning contract and observed/inferred/estimated/unknown discipline.
3. `privacy-build-borrow-and-inference.md`
   - Privacy-first OSS architecture rules, build-vs-borrow decisions, local-inference non-canonical policy.
4. `verification-and-release-gates.md`
   - Verification matrix mapping, release-readiness checklist, doc/claim drift checks.

## Usage pattern
1. Select applicable skill(s) for the change.
2. Execute each skill checklist and attach evidence.
3. Update docs/tests/contracts where a checklist fails.
4. Record residual risk explicitly.

These skills augment (not replace) `AGENTS.md` and `docs/repo-os/model-spec.md`.
