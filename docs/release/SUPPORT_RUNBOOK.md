# Support Runbook

## First response checklist

1. Collect MEL version/build info.
2. Run `mel doctor` and capture output.
3. Export support bundle.
4. Determine if issue is known limitation vs defect.
5. Request timeline/action IDs for incident window.

## Support asks first

- Configuration file (sanitized)
- Doctor output
- Transport status snapshot
- Relevant incident/action IDs
- Proofpack/support-bundle artifact

## Escalation criteria

Escalate when:

- issue contradicts documented support matrix
- deterministic evidence suggests data corruption
- action governance/audit trails are inconsistent
