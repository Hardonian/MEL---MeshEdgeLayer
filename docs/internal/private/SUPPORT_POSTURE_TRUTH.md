# Support and SLA posture — truth doc (internal)

## What the OSS project commits to

- **Community best effort** on issues and discussions — no guaranteed response time.
- **Security reporting** path in [SECURITY.md](../../../SECURITY.md).
- **Accuracy expectation:** docs and UI should match implemented behavior; gaps are bugs or documentation defects.

## What is not committed without a separate agreement

- 24/7 on-call, fixed MTTR, or severity-based SLAs.
- Legal indemnity, compliance certifications, or pen-test attestation “by default.”
- Hosted multi-tenant operations run by the core OSS maintainers unless explicitly offered elsewhere.

## Paid or pilot support (if offered)

Any future commercial support must reference **concrete** deliverables: response windows, channels, escalation path, data handling, and version pins. Do not imply SLAs from this repository alone.
