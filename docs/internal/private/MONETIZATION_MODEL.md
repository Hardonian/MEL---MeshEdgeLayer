# Monetization Model

## Valid paths

1. Paid deployment support and priority response.
2. Paid operational enablement (runbook setup, incident drills, release readiness).
3. Paid advanced governance/incident-memory features as they ship.

## Invalid paths

- Mandatory cloud lock-in for core MEL viability.
- Selling unsupported transport promises.
- Packaging that hides degraded-state reality.

## Near-term revenue mix (target)

- 50–70% services/support
- 20–40% subscription packaging
- 0–10% design-partner funded roadmap acceleration
