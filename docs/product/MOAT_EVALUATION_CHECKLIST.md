# Moat Evaluation Checklist

Use this before major roadmap or packaging decisions.

- [ ] Did this change improve incident/action/evidence memory quality?
- [ ] Does the capability compound with repeated operational usage?
- [ ] Can the resulting value be replicated from UI screenshots alone?
- [ ] Did we strengthen operator trust (less ambiguity, better auditability)?
- [ ] Did we preserve privacy-first and local-first constraints?
- [ ] Are recommendation outputs explicitly bounded by uncertainty?
- [ ] Is there a deterministic verification path for the claims?

Classification guidance:
- **Maintenance:** correctness/reliability without compounding advantage.
- **Leverage:** faster safer operator workflows with measurable trust gain.
- **Moat:** durable compounding advantage from MEL-specific evidence memory.
