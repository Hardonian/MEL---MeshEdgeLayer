# Incident Memory Strategy

## Goal

Turn incident handling into reusable institutional memory without overstating certainty.

## Memory objects that should compound

- Incident signatures and recurring-failure fingerprints.
- Operator adjudication notes and rationale.
- Action outcome history by context (transport, topology, constraints).
- Proofpack completeness and evidence sufficiency markers.

## Minimum data discipline

- Preserve actor attribution and timestamps for every control transition.
- Keep observed vs inferred fields separate.
- Tag unresolved uncertainty explicitly instead of forcing closure.

## Reuse loop

1. Incident triage produces structured evidence.
2. Decisions and outcomes are recorded.
3. Patterns are reviewed for runbook updates.
4. Updated runbooks improve future incident response.
