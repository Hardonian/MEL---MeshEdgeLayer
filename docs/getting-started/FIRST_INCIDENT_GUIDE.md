# First Incident Guide

## Step 1: establish truth boundaries

- Is this live data, stale data, or missing data?
- Which transports are actually ingesting now?

## Step 2: inspect incident details

- Review evidence fields before recommendations.
- Mark unknowns explicitly.

## Step 3: evaluate actions safely

- Confirm whether action is advisory-only or executable.
- Apply approval controls for higher-risk actions.

## Step 4: preserve evidence

- Add operator notes.
- Export proofpack/support bundle.
- Capture incident outcome and follow-up tasks.
