# Topology scaffolds

This tree reserves repo-local space for node-role-specific assets that sit beside MEL's core runtime.

- `central-node/` is for hub-side deployment assets that support MEL as the aggregation and observability layer.
- `extension-node/` is for constrained device assets that feed data toward a central MEL deployment.

These directories are intentionally documentation-first until code or verified assets exist. They must not be used to imply implemented runtime support that MEL does not currently ship.
