# MEL Code of Conduct

## Purpose

MEL — MeshEdgeLayer — is a technically serious, evidence-driven project. We are here to build professional, local-first observability systems for Meshtastic users. To maintain the project's integrity, we follow a simple code of conduct.

## Our Standards

- **Be Respectful**: Treat contributors and operators with professional courtesy.
- **Be Evidence-Driven**: Claims about transport support, hardware validation, or performance must be backed by data, logs, or repo-local tests.
- **Be Honest about Limits**: Never overstate Mel's current capabilities. If a transport is experimental or a feature is only a scaffold, document it as such.
- **Zero Theatre**: Avoid marketing fluff. Stick to technical truth and operator outcomes.
- **Quiet Complexity**: Aim to reduce entropy. Do not introduce ornamental noise or unneeded dependencies.

## Scope

This Code of Conduct applies within all project spaces, and it also applies when an individual is representing the project or its community in public spaces.

The project is governed by the the principle: **Does this reduce entropy or increase it?**

## Enforcement

Maintainers will follow up on reports of behavior that violates these principles. Actions range from private follow-up to removal from project spaces.

In instances of technical disagreement, the **canonical execution contract** (in `docs/roadmap/`) and **MEL Agent Notes** (`AGENTS.md`) serve as the source of truth for architectural decisions.

We are all here to make meshes more observable and reliable. Let's keep the focus on that.
